# revisions

所有需求文档、开发计划、设计方案、重构计划统一存放于此。

## 目录结构

```
revisions/
├── INDEX.md              ← 完整索引（入口）
├── README.md             ← 本文件
└── <功能名>-<日期>/      ← 按功能/主题的子目录
    ├── README.md         ← 该修订的 scope 说明
    └── ...               ← 需求文档、设计方案等
```

## 使用方式

- **浏览所有修订** → 看 `INDEX.md`
- **按功能查找** → 进对应子目录
- **新建修订** → 创建 `<功能名>-<日期>/` 子目录 + 更新 `INDEX.md`
